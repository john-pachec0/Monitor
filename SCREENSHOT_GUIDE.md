**Last Updated:** November 10, 2024

This guide explains what screenshots to create and how to create them for the App Store listing.

## How Many Screenshots?

- **Minimum:** 3 screenshots (Apple requirement)
- **Maximum:** 10 screenshots
- **Recommended:** 6-8 screenshots

**Why 6-8?**
- Enough to show key features
- Not so many users get bored
- Covers the main user journey

## Screenshot Content (In Order)

### Screenshot 1: Hero - Privacy Focus
**Purpose:** Lead with unique selling proposition (privacy)

**What to show:**
- Home screen or a custom graphic
- Large text overlay: "Your Data Stays on Your Device"
- Subtext: "No accounts â€¢ No cloud â€¢ No tracking"
- App name/icon visible

**Caption:** "Complete privacy for your mental health data"

**Design notes:**
- This is the first thing users see
- Make privacy message crystal clear
- Use warm, calming colors (match app theme)
- Device frame: Yes

---

### Screenshot 2: Quick Capture
**Purpose:** Show ease of capturing anxious thoughts

**What to show:**
- Capture thought screen
- Keyboard visible (shows it's a text entry)
- Example thought partially entered: "I'm going to mess up the presentation..."
- Clean, minimal interface

**Caption:** "Capture anxious thoughts in seconds"

**Design notes:**
- Show it's quick and simple
- Example thought should be relatable
- Don't make the example too intense/triggering
- Device frame: Yes

---

### Screenshot 3: Worry Box
**Purpose:** Show where thoughts are stored

**What to show:**
- Worry Box view with 3-4 captured thoughts
- Thoughts with timestamps showing "2 hours ago", "Yesterday", etc.
- Visual indication of thoughts ready for review
- Badge or indicator showing review time

**Caption:** "Review your worries when the time is right"

**Design notes:**
- Show multiple thoughts (not just one)
- Demonstrates the "postpone review" concept
- Thoughts should look like cards/containers
- Device frame: Yes

---

### Screenshot 4: Guided Review Process
**Purpose:** Show CBT methodology in action

**What to show:**
- Review screen showing distortion selection
- List of cognitive distortions (All-or-Nothing, Catastrophizing, etc.)
- Progress indicator (Step 2 of 5)
- One or two distortions selected/highlighted

**Caption:** "Identify thinking patterns with evidence-based CBT"

**Design notes:**
- Shows structured, professional approach
- Not overwhelming despite multiple options
- Progress indicator reassures users
- Device frame: Yes

---

### Screenshot 5: Reframe Creation
**Purpose:** Show the core value - thought reframing

**What to show:**
- Reframe editor with example reframe partially written
- Original thought visible at top for context
- Example reframe: "Even if I stumble, it doesn't mean I'll fail. I've prepared well..."
- Helpful placeholder text visible

**Caption:** "Create balanced, realistic perspectives"

**Design notes:**
- Show before/after thought transformation
- Example should demonstrate the reframing concept
- Warm, supportive UI colors
- Device frame: Yes

---

### Screenshot 6: Learn Section
**Purpose:** Show educational value

**What to show:**
- Learn home screen with cognitive distortion cards
- Cards showing icons and names (All-or-Nothing, Overgeneralization, etc.)
- Clean, organized grid or list
- Indicates ~3 min read time on cards

**Caption:** "Understand the 13 cognitive distortions"

**Design notes:**
- Shows depth of educational content
- Demonstrates evidence-based approach
- Cards should look tappable/interactive
- Device frame: Yes

---

### Screenshot 7: Pattern Recognition (Optional)
**Purpose:** Show long-term value

**What to show:**
- Archive or history view
- Thoughts organized by date/week
- Visual indication of patterns (maybe grouped by distortion type)
- Shows app has lasting value

**Caption:** "Notice patterns and track your progress"

**Design notes:**
- Shows the app is useful over time
- Demonstrates ongoing value
- Not critical if you have 6 screenshots already
- Device frame: Yes

---

### Screenshot 8: Settings/Customization (Optional)
**Purpose:** Show control and flexibility

**What to show:**
- Settings screen
- Worry time customization
- Notification options
- Privacy-focused settings (no accounts, local only)

**Caption:** "Customize your worry time and preferences"

**Design notes:**
- Shows user has control
- Reinforces privacy (no account settings)
- Only if you have room for 8 screenshots
- Device frame: Yes

## Design Specifications

### Device Frames
**Use device frames:** Yes, makes screenshots look more professional

**Options:**
- Use iPhone mockup templates (search "iPhone mockup PSD")
- Use Screenshot Maker apps (like Previewed, App Store Screenshot Maker)
- Use Figma/Sketch with device frame plugins

### Text Overlays
**Font:** SF Pro (iOS system font) or similar clean sans-serif

**Caption placement:**
- Below the device frame
- Or as overlay at top/bottom of device screen
- Keep captions short (5-10 words max)

**Text color:**
- Dark text on light background
- Or light text on semi-transparent dark overlay
- Ensure high contrast (WCAG AA minimum)

### Colors
**Background:**
- Match app theme (warm terracotta/peach tones)
- Or use soft gradient
- Or keep it white/light for clarity

**Consistency:**
- All screenshots should have same style
- Same device frame
- Same caption styling
- Same background treatment

### Content Guidelines
**Example content:**
- Use realistic but non-triggering examples
- Avoid mentioning specific diagnoses
- Keep examples general (work stress, social anxiety)
- Nothing too dark or heavy

**Things to avoid:**
- Self-harm references
- Specific mental health diagnoses
- Overly intense or triggering content
- Anything that violates App Store guidelines

## How to Create Screenshots

### Method 1: iPhone Simulator (Easiest)
1. Build and run app in iPhone 15 Pro Max simulator
2. Navigate to each screen you want to capture
3. **Simulator â†’ File â†’ Save Screen** (or Cmd+S)
4. Repeat for iPhone 8 Plus simulator
5. Add device frames and captions in design tool

### Method 2: Physical Device (Most Accurate)
1. Build and run on iPhone 15 Pro Max or similar
2. Navigate to screens
3. Take screenshots (Volume Up + Side button)
4. AirDrop to Mac
5. Add device frames and captions

### Method 3: Design Tool (Most Professional)
1. Export screens from Figma/Sketch design files
2. Or recreate screens in design tool
3. Use device mockup templates
4. Add frames and captions all in design tool
5. Export at correct resolutions

**Recommended:** Method 1 (simulator) for speed, Method 3 for perfection

## Screenshot Checklist

Before uploading to App Store Connect:

### Technical
- [ ] Correct resolutions (1290x2796 for 6.7", 1242x2208 for 5.5")
- [ ] PNG or JPEG format (PNG recommended)
- [ ] File size under 10MB each
- [ ] No transparent backgrounds
- [ ] Not stretched or pixelated

### Content
- [ ] No profanity or inappropriate content
- [ ] No references to other platforms (Android, etc.)
- [ ] No pricing or promotional text (unless in "What's New")
- [ ] Accurate representation of app (no fake features)
- [ ] All text is readable at thumbnail size

### Design
- [ ] Consistent style across all screenshots
- [ ] Same device frame (if using)
- [ ] Captions are clear and concise
- [ ] High contrast, readable text
- [ ] Professional appearance

### Examples
- [ ] Example thoughts are appropriate
- [ ] Not too triggering or intense
- [ ] Relatable to target audience
- [ ] Shows real app functionality

## Order Matters

**Most important screenshot:** First one (users see this in search results)

**Sequence should tell a story:**
1. **Hook** - Privacy promise (unique value)
2. **Capture** - Show ease of use
3. **Store** - Show worry box concept
4. **Review** - Show CBT process
5. **Reframe** - Show transformation
6. **Learn** - Show depth
7. **Track** - Show long-term value (optional)
8. **Customize** - Show control (optional)

**Users decide in 3-5 screenshots** whether to download, so front-load the most important ones.

---

## Examples to Avoid

**DON'T:**
- Show empty screens (looks incomplete)
- Use Lorem Ipsum or placeholder text
- Show debug menus or test data
- Include version numbers or "beta" labels
- Show error messages or bugs
- Use low-quality images
- Violate Apple's screenshot guidelines

**DO:**
- Show realistic use cases
- Use clean, high-quality renders
- Highlight unique features (privacy!)
- Make text readable even at small sizes
- Test how they look in App Store search (thumbnail view)

## Testing Your Screenshots

Before finalizing:

1. **View at thumbnail size**
   - First screenshot appears tiny in search results
   - Can you tell what the app does?
   - Is privacy message clear?

2. **Show to someone unfamiliar**
   - Do they understand what the app does?
   - Does it look trustworthy?
   - Would they download it?

3. **Check on different devices**
   - How does it look on iPad? (if offering iPad version)
   - How does it look on Mac App Store?

4. **Compare to competitors**
   - How do your screenshots compare to other CBT apps?
   - Do you stand out?
   - Is privacy message prominent?

## Updating Screenshots

You can update screenshots anytime without submitting a new app version.

**When to update:**
- After significant UI changes
- When adding new features
- If user feedback suggests confusion
- To test different messaging (A/B testing with phased releases)

**Best practice:** Keep screenshots current with latest app version
